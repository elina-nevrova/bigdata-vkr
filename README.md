**Тема:** Создание облачного сервиса для расчёта экономической эффективности и формирования рекомендаций по изделиям для 3D-печати

**Продукт:** Облачный сервис для владельцев 3D-ферм — селлеров на маркетплейсах (расчёт экономической эффективности изделий, аналитика рынка, ML-рекомендации по нишам).

## Архитектура

Сервис построен из слабосвязанных слоёв: сбор → хранение → обработка/ETL → ML → API → UI.

```
[Wildberries]
        │  (Scrapy / Playwright)
        ▼
[Data Collection Layer] ──► [PostgreSQL]  (оперативные/транзакционные данные)
        │                        │
        │                        ▼
        └──────────────► [ClickHouse]  (аналитические витрины)
                                 │
                                 ▼
              [Airflow ETL: нормализация, метрики]
                                 │
                    ┌────────────┴────────────┐
                    ▼                         ▼
              [ML-слой: прогноз спроса   [FastAPI backend]
               + рекомендатор]                 │
                    └────────────┬────────────┘
                                 ▼
                        [Vue.js дашборды]
```

## 1. Слой сбора данных (Data Collection Layer)

| Компонент | Технология | Назначение |
|---|---|---|
| Скрейпинг | **Python + Scrapy / Playwright / patchright** | Сбор карточек: SKU, категория, цена, рейтинг, отзывы, продавец, динамика |
| Источник | Wildberries | Карточки товаров и рыночные показатели |
| Планировщик | Cron / Airflow schedule | Ежедневное инкрементальное обновление |

**Целевой объём:** ≥ 10 категорий, ≥ 10 000 карточек, ≥ 100 000 записей рыночных показателей.

**Резервные варианты данных (для запуска / обучения):**
- Kaggle-датасеты по Wildberries (2022–2023) — для первичного обучения ML.
- GitHub-выборка ~1001 листинг WB (Bright Data) — демонстрация структуры данных.
- Apify / Chrome-расширения — быстрый сбор нескольких тысяч карточек без разработки.

## 2. Слой хранения (Storage Layer)

| СУБД | Роль |
|---|---|
| **PostgreSQL** | Транзакционные данные: профили пользователей, принтеры, материалы, сохранённые расчёты и сценарии |
| **ClickHouse** | Колоночное хранилище для аналитических запросов по рыночным показателям (агрегации, временные ряды) |

---

## 3. Слой обработки и аналитики (Analytics Layer)

| Компонент | Технология | Назначение |
|---|---|---|
| ETL-оркестрация | **Apache Airflow** | Пайплайн: нормализация, дедупликация, расчёт метрик |
| Метрики рынка | — | Оборачиваемость, доля рынка, индекс конкуренции, сезонность |

---

## 4. ML-слой

| Задача | Методы | Комментарий |
|---|---|---|
| Прогноз спроса / трендов | **Prophet**, **LightGBM / CatBoost**, (опц. PyTorch) | Временные ряды; для ВКР — сравнение алгоритмов по метрикам |
| Рекомендательная система | Коллаборативная фильтрация + производственный профиль пользователя | Ранжирование ≥ 20 изделий по прогнозной экономической эффективности |
| Переобучение | Еженедельно на накопленных данных | — |

**Метрики качества (для ВКР):** MAE / MAPE / RMSE для прогноза; Precision@K, NDCG для ранжирования ниш.

**Научная новизна:** адаптация предиктивной аналитики к специфике аддитивного производства (структура затрат, ограничения маркетплейса).

## 5. Бэкенд / API

| Компонент | Технология |
|---|---|
| REST API | **FastAPI (Python)** |
| Калькулятор экономической эффективности | Python-модуль: филамент, время печати, брак (5–10 %), амортизация, электроэнергия, упаковка, комиссии, логистика, хранение, продвижение |

**Требования к калькулятору:** профили принтеров (≥ 3), материалов (≥ 10); полная себестоимость, валовая прибыль, маржинальность, окупаемость тиража; расчёт сценария ≤ 10 сек; сценарии «что если».

## 7. Инфраструктура и DevOps

- **Docker** — контейнеризация всех сервисов.
- **GitHub** — приватный репозиторий (300+ коммитов прототипа).
- **Figma** — разработка и дизайн.
- Тестовый сервер: 4 vCPU / 8 ГБ RAM.
